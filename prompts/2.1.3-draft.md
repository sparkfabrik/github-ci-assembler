problema:
  Cioè il problema è come si aggiornano tutte le action tra pacchetti e a
  livello di proj cioé pkg_drupal che usa action di base per oidc deve dichiarare
  versione, quale? la versione di pkg_base installata nel project (che comunque
  rispetta quello che il `pkg_drupal` definisce come suo requirement nel suo
  composer.json.)

soluzione:
  Introdurre una keyword env solo per il file di configurazione dell'assembler
  che a differenza delle altre tipologie dei file dell'assembler (proj e pkg) contribuisca
  alla keyword root env dei file di workflow di GitHub Actions, quindi a definire
  variabili globali di ambiente di shell.
  Il file di configurazione dell'assembler vivrà dentro la fs-cli o il
  `pkg_composer_plugin` e sarà twiggato e sarà responsabilità del Composer
  Plugin, che è quello che conosce che versioni sono installate in ciascun
  progetto, di preparare queste variabili globali.
